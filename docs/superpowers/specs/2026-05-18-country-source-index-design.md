# Country Source Index — Design Spec

**Date:** 2026-05-18
**Status:** Approved

## Problem

Corpscout currently has adapters for 5 countries (GB, NO, DK, EE) and one partial stub for RS. There is no systematic record of what company data sources exist globally, which are free vs paid, or which support API access vs scraping. As new adapters are built, this knowledge is scattered and gets lost between sessions.

## Goal

Create a structured research folder in the corpscout repo covering the top 100 countries by GDP. Each country gets its own markdown file documenting every known company data source: official registries, commercial providers, and aggregators. Files are updatable as new sources are discovered and serve as the authoritative reference when deciding which adapter to build next.

## Folder Structure

```
corpscout/docs/sources/
├── README.md                    # Master index table (rank, country, free?, API?, adapter status)
└── countries/
    ├── us.md    # #1  United States
    ├── cn.md    # #2  China
    ├── de.md    # #3  Germany
    ├── jp.md    # #4  Japan
    ├── in.md    # #5  India
    ├── gb.md    # #6  United Kingdom      (adapter: companies_house)
    ├── fr.md    # #7  France
    ├── it.md    # #8  Italy
    ├── ca.md    # #9  Canada
    ├── br.md    # #10 Brazil
    └── ...      # 100 files total, named by ISO2 lowercase
```

Files are named by ISO2 code in lowercase. The GDP ranking follows IMF 2024 nominal GDP data.

## Per-Country File Template

Every country file follows this exact structure. Sections are added or omitted based on what sources exist — if a country has no commercial providers, that section is omitted entirely.

```markdown
# [Country Name] ([ISO2]) — GDP Rank #[N]

> **Summary:** One or two sentences on the best path to company data for this country.

## Official Registry

### [Registry Name]
| Field      | Detail |
|------------|--------|
| URL        | https://... |
| Access     | API / Bulk download / Scrape |
| Cost       | Free / Free (registration required) / Paid |
| Fields     | name, registration number, address, status, ... |
| Auth       | None / API key / OAuth / Registration |
| Rate limit | N/A or specifics |
| Notes      | Any caveats, update frequency, data quality notes |

## Commercial Providers

### [Provider Name]
| Field      | Detail |
|------------|--------|
| URL        | https://... |
| Access     | API (paid) |
| Cost       | Paid — [price if known, else "contact for pricing"] |
| Fields     | ... |
| Auth       | API key |
| Rate limit | Per contract |
| Notes      | ... |

## Aggregators

### OpenCorporates
| Field  | Detail |
|--------|--------|
| Access | API (paid) |
| Cost   | Paid |
| Fields | name, number, status, address, incorporation date |
| Notes  | Note whether OC scrapes the official registry directly |

### GLEIF
| Field  | Detail |
|--------|--------|
| Access | API (free) |
| Cost   | Free |
| Fields | LEI, legal name, HQ country, parent LEI |
| Notes  | Legal entities with LEI only — large/listed companies |

## Corpscout Status
- [ ] Adapter implemented
- Source name: `—`
- Recommended source: [which source to build against and why]
- Priority: High / Medium / Low
```

### Access Values

| Value | Meaning |
|-------|---------|
| `API (free)` | REST/SOAP API, no cost |
| `API (paid)` | REST/SOAP API, requires payment |
| `Bulk download` | Static file download (CSV/JSON/XML), no cost |
| `Bulk download (paid)` | Static file download, requires payment |
| `Scrape` | No structured access — web scraping required |
| `Free (registration required)` | Free but requires account creation |

## Research Tiers

Countries are researched in three tiers during implementation:

| Tier | Ranks | Approach |
|------|-------|----------|
| 1 | 1–20 | Full research — highest value markets |
| 2 | 21–60 | Best-effort — key sources identified |
| 3 | 61–100 | Known sources filled in, stubs where unknown |

Countries already researched (GB, NO, DK, EE, RS) are pre-filled regardless of tier.

## Update Convention

When a new source is discovered for an existing country:
1. Add a new subsection under the appropriate heading in the country file
2. Update the `Corpscout Status` section if the recommended source changes
3. Update the README row if the "Best Free Option" or "API?" column changes

No approval process required — these are living research documents.
