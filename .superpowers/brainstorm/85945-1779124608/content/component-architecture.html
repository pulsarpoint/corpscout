Review Page — Component Architecture

Shared bulk-review shell wraps each tab. Tabs only provide columns + data + action handlers.

ui/app/routes/review.tsx
── routes/review.tsx ─── thin orchestrator, just tabs
📄 review.tsx
<Tabs>
  <TabsTrigger>Domain Candidates 25,288</TabsTrigger>
  <TabsTrigger>New Companies 0</TabsTrigger>
  // future tabs…
  <TabsContent> <DomainCandidatesTab /> </TabsContent>
  <TabsContent> <CompanySuggestionsTab /> </TabsContent>
</Tabs>
↓   ↓
📄 DomainCandidatesTab.tsx
Provides to BulkReviewTable:
columns = [company, domain,
          signal, confidence]
data = useReviewQueue()
onApprove = api.createReview(id, 'approved')
onReject = api.createReview(id, 'rejected')
filterFields = [signal, confidence]
📄 CompanySuggestionsTab.tsx
Provides to BulkReviewTable:
columns = [name, country,
          confidence, source]
data = useCompanySuggestions()
onApprove = api.approveSuggestion(id)
onReject = api.rejectSuggestion(id)
filterFields = [country, confidence]
↓
🧩 BulkReviewTable<TData> — SHARED COMPONENT
Props interface:
columns: ColumnDef<TData>[]
data: TData[]
total: number
loading: boolean
onApprove(ids: string[]): Promise<void>
onReject(ids: string[]): Promise<void>
filterDefs: FilterDef[]
onFilterChange(f): void
page, onPageChange
Owns internally:
• Row selection state (rowSelection)
• Bulk action bar (appears when >0 selected)
• Filter chip bar (active filters shown)
• "Select all on page" checkbox
• Column sorting state
• Detail sheet (onRowClick)
• Loading / empty states

Does this component split feel right? BulkReviewTable owns all selection/filter/bulk-action logic. Tabs are just columns + data + handlers.
