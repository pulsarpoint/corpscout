A — Tabs on existing page Recommended

Extend sources_.$name.tsx into a 4-tab page. Consistent with the rest of the UI (companies, jobs). Raw Inputs gets sub-tabs for Live / Archive.

Pros

  • Matches existing UI patterns
  • Single URL per source
  • Easiest to build

Cons

  • Raw Inputs tab is heavier than others

B — Left sidebar nav

Persistent sidebar within the source page. More space for content, familiar admin-panel feel. Each section has its own area.

Pros

  • More horizontal space per section
  • Sections always visible in nav

Cons

  • Different pattern from rest of UI
  • More complex layout

Deep-linkable URLs.

C — Separate sub-routes

Each section becomes its own route: /sources/gleif/schedule, /sources/gleif/config, etc. Maximum separation, fully deep-linkable.

Pros

  • Clean isolation per section
  • Deep-linkable

Cons

  • More routes/files to maintain
  • Context switches feel heavy
